Fall

unicode

Verbietet Schreibmaschinenlösungen wenn bessere Unicode‐Zeichen verfügbar sind.

Festlegung

case unicode

Einzelheiten

Diese Regel gilt nicht, wo bessere Zeichen nicht erkannt wären:

print("Hello, world!") // ← Erlaubt, weil Anführungszeichen hier nicht funktionieren.

In manche Zusammenhänge, wie beim Ersellung einer Parallelbezeichnung, sind markierte ausnahmen trotzdem nötig:

func ≠ (a: Any, b: Any) -> Bool { return a != b // @ausnahme(zu: unicode) }

Diese Regel fasst folgendes um:

  • Waagerechte Striche:

    • Bindestriche: „E‐Mail“ (U+2010) anstatt „E-Mail“ (U+002D).

    • Minuszeichen: „3 − 1 = 2“ (U+2212) anstatt „3 - 1 = 2“ (U+002D).

    • Gedankenstriche: „–“ anstatt „--“.

    • Aufzählungszeichen: „•“ anstatt „-“.

  • Hochgestellte Striche:

    • Anführungszeichen: „... ‚...‘ ...“ (U+2018–U+201F) anstatt "... '...' ..." (U+0022, U+0027).

    • Apostrophe: „die Grimm’schen Märchen“ (U+2019) anstatt „die Grimm'schen Märchen“ (U+0027).

    • Gradzeichen: „20 °C“ anstatt „20 'C“.

    • Prime symbols: 52° 31′ 00′′ N (U+2032) anstatt 52° 31' 00" N (U+0027, U+0022).

  • „⋅“ anstatt „*“.

  • „∶“ anstatt „/“.

  • „≠“ anstatt „!=“.

  • „≤“ anstatt „<=“.

  • „≥“ anstatt „>=“.

  • „¬“ anstatt „!“.

  • „∧“ anstatt „&&“.

  • „∨“ anstatt „||“.
